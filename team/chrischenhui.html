1. Overview of Dukemon

Figure 1. Dukemon's design.

Dukemon is a desktop flashcard application, whose aim is to streamline and gamify the process of learning words and definitions.
The user interacts with it primarily through a CLI (Command Line Interface), with some GUI (Graphical User Interface) features.

Allow me to brief you on some useful terms and to give you a context.

  1. Flashcards are termed as Cards.

  2. A collective set of Cards are termed as WordBank.

  3. A general flow of Dukemon's usage is as follows:

    1. Note taking phase

      1. User creates/selects a WordBank.

      2. User populates the WordBank with Cards, or edit the Cards.

    2. Learning phase

      1. User starts the game using selected WordBank.

      2. Dukemon prompts the questions, and user has to answer accordingly.

      3. Dukemon ends the game.

      4. User reviews performance from statistics page.

2. Summary of contributions

2.1. Major feature

Word Bank Management.

  • What it does:

    • Enables user to have a high level management for his word banks and cards through CLI and GUI.
      Is also semi-automated by Dukemon to enhance user’s learning experience.
      Consist of four complementary distinct sections.

      1. Word bank’s data structure and it’s storage system:
        Allows developers to use and extend this architecture to streamline their feature implementation.
        Is implemented with a robust underlying WordBankList data structure, with it’s own WordBankListStorage, along with rich integration of Model, Storage, Logic and UI to synchronise it’s data.

      2. User Commands:
        Allows users to edit cards/word banks through CardCommands / WordBankCommands.

      3. Drag and drop:
        Allows user to export their word bank to their computer simply by dragging it out of Dukemon.
        Likewise, it allows user to drag a word bank file into Dukemon to import the word bank.

      4. Revision word bank:
        Serves as a centralised bank that collects cards across different word banks throughout the game play, to enhance learning experience.
        Cards that were answered wrongly are automatically added to this revision bank.
        Likewise, cards that were answered correctly during game play are automatically removed from this revision bank.

  • Justification:

    • Management system is essential for a self-typed flashcard app like Dukemon.

    • Well designed architecture and well implemented data structure streamlines integration with other features that builds on top of it. (statistics, game, settings)

    • Accelerates sharing of word banks with friends, thereby enhancing learning experience through sharing of knowledge/notes with friends.

    • Revision bank improves learning experience for the user, by helping the user automatically aggregate cards that were answered wrongly.

  • Highlights:

    • As this feature lays the basis for the app, it needs to be implemented well to support integration with every component of the app.
